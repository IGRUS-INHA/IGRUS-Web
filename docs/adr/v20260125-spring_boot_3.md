# Spring Boot 버전 - Spring Boot 3 사용

## 선택지
1. Spring Boot 3.x: 안정화된 버전 사용
2. Spring Boot 4.x: 최신 버전 사용 (2026년 1월 기준 출시 초기)

## 결정
- Spring Boot 3.x 버전 채택

## 결정 이유

### 1. Spring Boot 4 출시 초기 불안정성
- 2026년 1월 기준 Spring Boot 4는 출시 초기 단계
- 새 메이저 버전 출시 직후에는 버그나 예상치 못한 이슈가 발생할 가능성이 높음
- 프로덕션 환경에서 안정성이 검증되지 않은 상태

### 2. 서드파티 라이브러리 호환성 문제
- 프로젝트에서 사용하는 주요 라이브러리들이 아직 Spring Boot 4를 정식 지원하지 않음
- Spring Security, Spring Data JPA 등 핵심 의존성의 4.x 호환 버전 부재 또는 불안정
- 테스트 라이브러리, 모니터링 도구 등 생태계 전반의 지원 미비

### 3. 문서 및 커뮤니티 지원
- Spring Boot 3.x는 풍부한 문서와 커뮤니티 자료가 축적되어 있음
- 문제 발생 시 해결책을 찾기 용이
- Spring Boot 4 관련 자료는 아직 부족한 상태

### 4. 동아리 프로젝트 특성
- 학기 중 개발 일정을 고려할 때 안정적인 환경이 우선
- 신규 버전 이슈 대응에 시간을 투자하기보다 기능 개발에 집중 필요

## 고려한 대안

### Spring Boot 4.x
**장점:**
- 최신 기능 및 성능 개선 사항 활용 가능
- Java 21 최적화 개선
- 장기적으로 더 오랜 지원 기간

**채택하지 않은 이유:**
- 출시 초기로 생태계 지원이 불충분
- 의존 라이브러리들의 호환 버전이 아직 안정화되지 않음
- 마이그레이션 가이드 및 사례가 부족하여 예상치 못한 이슈 해결이 어려움

## 결과

- 현재 프로젝트는 Spring Boot 3.x 버전을 기반으로 개발
- Spring Boot 4 생태계가 안정화되면 마이그레이션 진행 예정
- 마이그레이션 시점 판단 기준:
  - 주요 서드파티 라이브러리들의 Spring Boot 4 정식 지원
  - 커뮤니티에서 충분한 적용 사례 및 안정성 검증
  - Spring Boot 4.x 마이너 버전 2~3회 이상 릴리즈 후 검토
