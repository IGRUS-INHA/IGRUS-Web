# Architecture Decision Records (ADR)

이 폴더는 프로젝트의 주요 아키텍처 결정 사항을 기록합니다.

## ADR이란?

ADR(Architecture Decision Record)은 소프트웨어 아키텍처에 관한 중요한 결정을 문서화하는 방법입니다. 각 ADR은 특정 결정의 배경, 이유, 그리고 결과를 기록합니다.

## 파일 명명 규칙

```
v{YYYYMMDD}-{slug}.md
```

- `YYYYMMDD`: 결정 날짜
- `slug`: 결정 내용을 간략히 설명하는 식별자 (영문 소문자, 언더스코어 사용)

예시: `v20260121-kakao_login.md`

## 새 ADR 작성 가이드

새로운 아키텍처 결정을 기록할 때 다음 내용을 포함하세요:

1. **배경**: 결정이 필요했던 상황
2. **결정**: 어떤 결정을 내렸는지
3. **이유**: 왜 그런 결정을 내렸는지
4. **결과**: 결정으로 인한 영향 및 후속 조치
